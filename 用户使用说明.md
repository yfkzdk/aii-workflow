# AII上下文助手 - 用户使用说明

## 这个工具是做什么的？

一个基于 Claude Code 的自动化任务执行系统。你提供需求，系统自动：
1. 优化需求、生成多方案对比
2. 规划执行步骤
3. 生成代码并验证
4. 归档结果

---

## 快速开始

### 方式一：Python 测试入口

```bash
cd "O:\AII\上下文助手"
python tests/test_phase3.py
```

这会运行完整的端到端测试（mock 模式，不调用真实 LLM）。

### 方式二：Demo 演示

```bash
python demo/demo_phase3.py
```

演示完整管线流程，从输入收集到归档。

---

## 核心架构

```
用户输入 → Orchestrator (Python编排器)
         → AgentCaller (SDK/Fallback)
         → LLM 推理
         → Tool Use 审批
         → 验证 + 质量门
         → 下一阶段
```

### 8 个管线阶段

| 阶段 | Agent | 说明 |
|------|-------|------|
| input_collecting | input_collector | 收集用户需求 |
| requirement_optimizing | requirement_optimizer | 生成方案对比 |
| confirmation | (用户确认) | 选择方案 |
| planning | planner | 生成执行计划 |
| prompt_optimizing | prompt_optimizer | 优化提示词 |
| executing | coder | 生成代码 |
| verifying | verifier | 运行验证 |
| archiving | archivist | 归档结果 |

## 核心文件

| 文件/目录 | 作用 |
|----------|------|
| `core/` | 核心模块（StateDB, Orchestrator, AgentCaller） |
| `scripts/` | 工具脚本（validator, requirement_optimizer, utils） |
| `tests/` | 测试套件 |
| `demo/` | 演示脚本 |
| `.claude/agents/` | Agent 定义文件 |
| `config/` | 配置文件（skills.json, requirement_schema.json） |
| `workflows/` | 任务数据目录 |

---

## 状态存储

任务状态存储在 `workflows/<task-id>/state.db`（SQLite WAL 模式）。

关键表：
- `task_state`: 任务主状态（status, step_index, user_input_json, confirmation_json）
- `snapshots`: 快照/回滚支持

---

## 如何运行真实任务

目前系统处于 **mock 测试阶段**，尚未接入真实 LLM。

要接入真实 LLM，需要：
1. 配置 `ANTHROPIC_API_KEY` 环境变量
2. AgentCaller 会自动使用 SDK 模式
3. 或使用 Fallback 模式调用本地 `claude` CLI

---

## 故障恢复

系统支持：
- **自动重试**：验证失败时自动回退并重试（最多 3 次）
- **快照回滚**：每个阶段前保存快照，可手动回滚
- **断点续跑**：中断后重新运行 `run()` 从上次状态继续

---

## 注意事项

- 任务描述要**简短明确**，3-5 句话最好
- 大任务建议拆成小任务逐个执行
- 当前版本为 **v0.5.0**，核心架构已完成，待接入真实 LLM

---

## 开发者入口

```python
from core import StateDB, Orchestrator, AgentCaller

# 初始化任务
db = StateDB("workflows/TEST-001")
db.init_task("TEST-001")

# 运行编排器
orch = Orchestrator("workflows/TEST-001", "TEST-001")
orch.handle_user_input("帮我写一个排序算法")
result = orch.run()
```
