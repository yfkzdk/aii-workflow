```mermaid
sequenceDiagram

    participant O as Orchestrator
    participant A as AgentCaller
    participant L as LLM
    participant V as Validator
    participant D as StateDB

    U->>O: 提交需求
    O->>D: init_task()
    D-->>O: status=input_collecting

    Note over O: 阶段1: input_collecting
    O->>A: call(input_collector)
    A->>L: 推理请求
    L-->>A: 结构化输入
    A-->>O: 用户输入chunks
    O->>D: save_user_input()

    Note over O: 阶段2: requirement_optimizing
    O->>D: save_snapshot()
    O->>A: call(requirement_optimizer)
    A->>L: 生成方案对比
    L-->>A: proposals JSON
    A-->>O: 优化需求
    O->>D: update_status(confirmation)

    Note over O: 阶段3: confirmation
    U->>O: 选择方案
    O->>D: save_confirmation()

    Note over O: 阶段4: planning
    O->>A: call(planner)
    A->>L: 生成执行计划
    L-->>A: plan JSON
    A-->>O: 任务分解

    Note over O: 阶段5: prompt_optimizing
    O->>A: call(prompt_optimizer)
    A->>L: 优化提示词
    L-->>A: optimal_prompt.md
    A-->>O: 提示词优化

    Note over O: 阶段6: executing
    O->>A: call(coder)
    A->>L: 生成代码
    L-->>A: code files
    A-->>O: 代码实现

    Note over O: 阶段7: verifying
    O->>V: validate_step()
    V->>V: py_compile / JSON schema 校验
    V-->>O: PASS / FAIL

    alt 验证通过
        O->>D: update_status(archiving)
    else 验证失败
        O->>D: increment_retry()
        O->>D: save_snapshot()
        Note over O: 补偿回退 + 重试 (最多 max_retries 次)
    end

    Note over O: 阶段8: archiving
    O->>A: call(archivist)
    A->>L: 生成归档报告
    L-->>A: AI_WORKFLOW_LOG.md
    A-->>O: 归档完成
    O->>D: update_status(completed)
    O-->>U: 任务完成
```

```mermaid
    Transient -->|No| Stop

    MaxRetry -->|No| Failed[标记 failed]

    Validation --> Compensate{补偿回退}
    Compensate -->|verifying| BackToExec[回退 executing]

    Compensate -->|其他| StayStage[保持当前步骤重试]
```

## 架构说明

### 核心设计原则

1. **分层职责分离**: Orchestrator (编排) → AgentCaller (调用) → LLM (推理)
2. **确定性执行**: 状态驱动，快照回滚
3. **Fail-Fast 拦截**: 验证器语法检查 + 重试耗尽检测
4. **补偿回退**: verifying → executing → prompt_optimizing 逐级回退

### 关键组件

- **Orchestrator**: Python 编排器，驱动流水线流转，retry/compensation 管理
- **StateDB**: SQLite WAL 状态存储，支持快照/回滚
- **AgentCaller**: OpenAI/Anthropic/Fallback 多模式调用器
- **Validator**: 步骤输出验证（JSON schema / py_compile / 文件存在检查）

### 数据持久化

- **state.db**: 任务状态 (SQLite WAL)
- **artifacts/**: 阶段产物 (JSON/MD/PY)
- **snapshots/**: 快照备份（snapshots 表，内置于 state.db）
