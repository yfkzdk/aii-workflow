# 🖥️ 上下文助手 - 用户界面使用指南

## 📋 界面选项概览

根据您的使用习惯，可以选择以下任意一种界面：

### 1. **菜单驱动界面** (推荐新手)
- 文件：`user_interface.py`
- 启动：`python user_interface.py`
- 特点：数字菜单选择，无需记忆命令

### 2. **Windows批处理界面** (Windows用户)
- 文件：`user_interface.bat`
- 启动：双击文件或在命令行运行
- 特点：原生Windows支持，无需Python知识

### 3. **命令行界面** (高级用户)
- 文件：`ww_simple.py`
- 启动：`python ww_simple.py "任务描述"`
- 特点：功能最全，灵活性最高

### 4. **快捷启动器** (最方便)
- 文件：`quick_launcher.sh` (bash) 或创建快捷方式
- 启动：双击运行
- 特点：一键选择使用方式

## 🚀 快速开始

### 对于Windows用户：
1. **最简单方法**：双击 `user_interface.bat`
2. **备用方法**：运行 `python user_interface.py`

### 对于Mac/Linux用户：
1. 运行 `python user_interface.py`
2. 或运行 `bash quick_launcher.sh`

## 🎯 核心功能说明

### 主菜单功能：
1. **📝 开始新任务**
   - 输入您的需求，如："请帮我分析这个项目的结构"
   - 系统会自动处理并给出结果

2. **📊 查看系统状态**
   - 检查助手当前状态
   - 查看可用工具和配置

3. **🔄 恢复中断的任务**
   - 如果上次任务中断，可以恢复继续

4. **ℹ️ 显示版本信息**
   - 查看系统版本和更新信息

5. **🧪 运行系统测试**
   - 验证所有功能是否正常
   - 生成测试报告

6. **❓ 显示帮助**
   - 查看详细使用说明

## 💡 使用技巧

### 任务描述建议：
- **具体明确**：不要只说"帮忙"，要说具体需求
- **包含上下文**：如果有相关文件或代码，可以提及
- **分步骤**：复杂任务可以分步请求

### 示例任务：
1. "请帮我分析这个Python项目的结构，找出主要模块"
2. "帮我创建一个用户注册的API端点"
3. "检查这个代码的性能问题并优化"
4. "写一个README文件介绍这个项目"

## 🔄 编码问题处理

如果在Windows上看到乱码：
1. 使用 `cn_wrapper.py` 包装器：`python cn_wrapper.py ww_simple.py "任务"`
2. 或者使用批处理界面，它会自动处理编码

## 📊 系统测试验证

在开始重要任务前，建议：
1. 运行"系统测试"验证功能
2. 查看生成的测试报告
3. 确保所有依赖正常

## 🎨 自定义界面

如果您想自定义界面：
1. 修改 `user_interface.py` 文件
2. 添加自定义功能
3. 调整菜单布局和选项

## 📞 技术支持

### 常见问题：
1. **界面不显示中文**：运行 `python fix_encoding_issues.py`
2. **找不到命令**：确保在正确目录运行
3. **功能异常**：运行系统测试检查问题

### 详细日志：
- 查看 `logs/` 目录中的日志文件
- 运行测试查看详细报告

## 🚀 高级用法

### 直接命令行使用：
```bash
# 基本任务
python ww_simple.py "请帮我写一个数据库迁移脚本"

# 系统操作
python ww_simple.py status
python ww_simple.py recover

# 使用编码包装器（解决Windows编码问题）
python cn_wrapper.py ww_simple.py "任务描述"
```

### 批处理文件集成：
可以将常用任务保存为批处理文件：
```batch
@echo off
python cn_wrapper.py ww_simple.py "每日代码审查"
pause
```

## 🎉 开始使用

### 第一步：验证系统
```bash
python user_interface.py
选择选项 5 运行系统测试
```

### 第二步：简单任务
```bash
选择选项 1
输入："请帮我介绍一下这个项目"
```

### 第三步：正式工作
```bash
选择选项 1
输入您的具体工作任务
```

---

**提示**：第一次使用时，建议先运行系统测试，确保所有功能正常。然后从简单任务开始，逐渐尝试复杂需求。

界面设计目标：**让非技术用户也能轻松使用强大的上下文助手功能！**
