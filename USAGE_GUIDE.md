## 🔄 **400错误中断后的恢复流程**

### **情况1：轻微中断（首次400错误）**
系统会自动处理：
1. 识别400错误
2. 重试计数+1
3. 清理上下文，重新开始
4. 继续执行

**你不需要做任何事情**，系统会自动重试（最多3次）。

### **情况2：严重中断（需要人工干预）**
当系统重试3次都失败后，你需要：

**步骤1：打开新的Claude Code窗口**
- 关闭当前的Claude Code会话
- 打开全新的Claude Code窗口（避免上下文污染）

**步骤2：运行恢复工具**
```bash
cd "O:\AII\上下文助手"
workflow_manager.bat
# 选择 [3] 恢复中断的工作流
```

**步骤3：选择要恢复的工作流**
系统会列出所有中断的工作流，选择你要恢复的那个。

**步骤4：复制恢复指令到新窗口**
系统会生成类似这样的指令：
```markdown
🤖 **恢复AII工作流 - 从断点继续**

工作流路径：O:\AII\上下文助手

任务ID：TASK-20260415-120000
当前状态：executing
重试次数：1/3

恢复指令：
1. 读取中断点状态文件：workflows/TASK-20260415-120000/state.json
2. 清空当前Claude Code对话历史（避免400错误）
3. 从断点处继续执行工作流
4. 使用简化的上下文重新开始
```

**步骤5：粘贴并发送**
把上面的指令复制到新的Claude Code窗口，按回车发送。

**步骤6：监控恢复进度**
运行 `workflow_manager.bat` 选择 [2] 查看状态。

## 💡 **最佳实践**

### **避免400错误的技巧**
1. **任务描述要简洁**：控制在3-5句话内
2. **复杂任务要拆分**：大任务拆成多个小任务
3. **利用历史日志**：系统会学习过去的成功模式
4. **及时清理**：定期清理已完成的工作流

### **提高成功率的方法**
1. **使用管理器启动**：`workflow_manager.bat` 会自动优化格式
2. **监控执行状态**：定期查看工作流状态
3. **利用中断恢复**：400错误不是失败，是机会
4. **查看历史模式**：在AI_WORKFLOW_LOG.md中学习

## 🚨 **常见问题解答**

### **Q: 如何知道工作流是否中断了？**
A: 运行 `workflow_manager.bat` 选择 [2]，如果有状态不是"completed"的任务，就是中断了。

### **Q: 中断恢复会丢失进度吗？**
A: **不会**。所有状态都保存在 `workflows/[任务ID]/` 目录中，恢复时会从断点继续。

### **Q: 最多可以重试多少次？**
A: 每个工作流最多重试3次。如果3次都失败，需要手动处理。

### **Q: 结果在哪里查看？**
A: 最终结果在 `tasks/output_result.md`，详细记录在 `workflows/[任务ID]/` 目录。

### **Q: 如何清理旧的任务？**
A: 运行 `workflow_manager.bat` 选择 [5]，会删除所有已完成的工作流目录。
